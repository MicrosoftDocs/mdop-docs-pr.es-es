# <span data-ttu-id="626b6-103">Novedades de MED-V 2.0</span><span class="sxs-lookup"><span data-stu-id="626b6-103">What's New in MED-V 2.0</span></span>


<span data-ttu-id="626b6-104">Microsoft Enterprise Desktop Virtualization (MED-V) 2,0 ha evolucionado la compatibilidad de la compatibilidad de aplicaciones para Windows 7 y ha quitado la funcionalidad que no es necesaria para este escenario.</span><span class="sxs-lookup"><span data-stu-id="626b6-104">Microsoft Enterprise Desktop Virtualization (MED-V) 2.0 has evolved the application compatibility support for Windows 7 and removed functionality that is not required for this scenario.</span></span> <span data-ttu-id="626b6-105">Por ejemplo, se han quitado características como el cifrado del área de trabajo de MED-V, el servidor MED-V centralizado y la transferencia de recorte de área de trabajo de MED-V.</span><span class="sxs-lookup"><span data-stu-id="626b6-105">For example, features such as encryption of the MED-V workspace, the centralized MED-V server, and MED-V workspace trim transfer have been removed.</span></span>

## <span data-ttu-id="626b6-106">Cambios en la funcionalidad estándar</span><span class="sxs-lookup"><span data-stu-id="626b6-106">Changes in Standard Functionality</span></span>


<span data-ttu-id="626b6-107">En esta sección se describen las áreas clave donde ha cambiado la funcionalidad de MED-V 2,0.</span><span class="sxs-lookup"><span data-stu-id="626b6-107">This section discusses the key areas where MED-V 2.0 functionality has changed.</span></span>

### <span data-ttu-id="626b6-108">Creación de áreas de trabajo de MED-V</span><span class="sxs-lookup"><span data-stu-id="626b6-108">MED-V Workspace Creation</span></span>

<span data-ttu-id="626b6-109">El disco duro virtual que se usa para el área de trabajo de MED-V ahora se crea en Windows Virtual PC.</span><span class="sxs-lookup"><span data-stu-id="626b6-109">The virtual hard disk used for the MED-V workspace is now created in Windows Virtual PC.</span></span> <span data-ttu-id="626b6-110">Entre los métodos que se usan para crear el área de trabajo MED-V se incluyen la instalación de Windows XP SP3, la actualización del sistema operativo y la preparación para su administración a través de la infraestructura de administración de software.</span><span class="sxs-lookup"><span data-stu-id="626b6-110">The methods that are used to create the MED-V workspace include installing Windows XP SP3, updating the operating system, and preparing it to be managed through software management infrastructure.</span></span>

<span data-ttu-id="626b6-111">Se han eliminado las funciones de administración sin conexión y transferencia de recorte, además de la funcionalidad propietaria de cifrado y compresión del área de trabajo de MED-V.</span><span class="sxs-lookup"><span data-stu-id="626b6-111">The offline management and trim transfer functionality were removed, in addition to the proprietary MED-V workspace encryption and compression functionality.</span></span> <span data-ttu-id="626b6-112">Al crear un área de trabajo de MED-V, un administrador debe preparar y configurar las aplicaciones y herramientas de administración apropiadas en la imagen en lugar de usar la herramienta de preparación de máquina virtual que se proporciona en MED-V 1,0.</span><span class="sxs-lookup"><span data-stu-id="626b6-112">When you create a MED-V workspace, an administrator should prepare and configure appropriate applications and management tools in the image instead of using the virtual machine preparation tool that is provided in MED-V 1.0.</span></span>

<span data-ttu-id="626b6-113">Ahora es necesario ejecutar Sysprep en la imagen de MED-V y validarlo durante el empaquetado del área de trabajo de MED-V.</span><span class="sxs-lookup"><span data-stu-id="626b6-113">Running Sysprep on the MED-V image is now required and validated during the packaging of the MED-V workspace.</span></span> <span data-ttu-id="626b6-114">El Empaquetador de área de trabajo MED-V proporciona una interfaz gráfica de usuario (GUI) que guía al administrador por el proceso de empaquetado.</span><span class="sxs-lookup"><span data-stu-id="626b6-114">The MED-V Workspace Packager provides a graphical user interface (GUI) that guides the administrator through the packaging process.</span></span> <span data-ttu-id="626b6-115">Se quitó la consola de MED-V 1,0 junto con la funcionalidad de administración de imágenes, la administración de perfiles del área de trabajo de MED-V y el requisito de ensayar y cifrar áreas de trabajo de MED-V.</span><span class="sxs-lookup"><span data-stu-id="626b6-115">The console from MED-V 1.0 was removed together with the functionality of managing images, managing MED-V workspace profiles, and the requirement to stage and encrypt MED-V workspaces.</span></span>

### <span data-ttu-id="626b6-116">Implementación del área de trabajo MED-V</span><span class="sxs-lookup"><span data-stu-id="626b6-116">MED-V Workspace Deployment</span></span>

<span data-ttu-id="626b6-117">Para implementar un área de trabajo de MED-V, un administrador puede aprovechar sus herramientas electrónicas de distribución de software.</span><span class="sxs-lookup"><span data-stu-id="626b6-117">To deploy a MED-V workspace, an administrator is now able to take advantage of their electronic software distribution tools.</span></span> <span data-ttu-id="626b6-118">El método de extracción de cliente disponible en MED-V 1,0 se quitó y el área de trabajo de MED-V se ha entregado usando métodos fuera de MED-V.</span><span class="sxs-lookup"><span data-stu-id="626b6-118">The client-pull method available in MED-V 1.0 was removed and the MED-V workspace is now delivered by using methods outside MED-V.</span></span> <span data-ttu-id="626b6-119">Los administradores pueden tratar áreas de trabajo de MED-V como cualquier otro paquete de aplicación y pueden programar implementaciones e instalaciones de MED-V con sus procesos y herramientas existentes.</span><span class="sxs-lookup"><span data-stu-id="626b6-119">Administrators can treat MED-V workspaces as they would any other application package and can schedule deployments and installations of MED-V by using their existing tools and processes.</span></span> <span data-ttu-id="626b6-120">Las instalaciones de MED-V se pueden implementar silenciosamente y se pueden administrar fácilmente dentro de una infraestructura de distribución de software existente.</span><span class="sxs-lookup"><span data-stu-id="626b6-120">MED-V installations can be deployed silently and can easily be managed inside an existing software distribution infrastructure.</span></span>

### <span data-ttu-id="626b6-121">Administración de áreas de trabajo de MED-V</span><span class="sxs-lookup"><span data-stu-id="626b6-121">MED-V Workspace Management</span></span>

<span data-ttu-id="626b6-122">El área de trabajo de MED-V en MED-V 2,0 se basa en un disco duro virtual de Windows Virtual PC.</span><span class="sxs-lookup"><span data-stu-id="626b6-122">The MED-V workspace in MED-V 2.0 is based on a Windows Virtual PC virtual hard disk.</span></span> <span data-ttu-id="626b6-123">MED-V ha ampliado las capacidades que proporciona Windows Virtual PC al mejorar la experiencia sin necesidad de cifrado o herramientas especiales para obtener acceso al área de trabajo de MED-V.</span><span class="sxs-lookup"><span data-stu-id="626b6-123">MED-V has extended the capabilities that Windows Virtual PC provides by improving the seamless experience without requiring encryption or special tools to access the MED-V workspace.</span></span>

<span data-ttu-id="626b6-124">Después de implementar MED-V en una estación de trabajo, el área de trabajo de MED-V se puede abrir en el modo de pantalla completa con Windows Virtual PC.</span><span class="sxs-lookup"><span data-stu-id="626b6-124">After MED-V is deployed to a workstation, the MED-V workspace can be opened in full-screen mode by using Windows Virtual PC.</span></span> <span data-ttu-id="626b6-125">Esta nueva funcionalidad ha quitado el requisito de las directivas que establecieron una preferencia para los modos de pantalla completa o transparente y también ha eliminado la necesidad de forzar la pantalla completa para el diagnóstico y la solución de problemas.</span><span class="sxs-lookup"><span data-stu-id="626b6-125">This new functionality removed the requirement for policies that set a preference for seamless or full-screen modes and also removed the need to force full-screen for diagnostics and troubleshooting.</span></span>

<span data-ttu-id="626b6-126">Las aplicaciones de publicación para el área de trabajo de MED-V ya no se realizan con perfiles y al escribir manualmente la ruta de acceso a las aplicaciones.</span><span class="sxs-lookup"><span data-stu-id="626b6-126">Publishing applications to the MED-V workspace is no longer performed with profiles and by manually entering the path to applications.</span></span> <span data-ttu-id="626b6-127">En su lugar, se produce de forma automática a medida que se instalan aplicaciones en el invitado.</span><span class="sxs-lookup"><span data-stu-id="626b6-127">Instead, it occurs automatically as applications are installed on the guest.</span></span> <span data-ttu-id="626b6-128">Se quita el repositorio de imágenes central que incluía las versiones de las imágenes que se entregaron a través de transferencia de recorte.</span><span class="sxs-lookup"><span data-stu-id="626b6-128">The central image repository that included versions of the images that were delivered through trim transfer is removed.</span></span> <span data-ttu-id="626b6-129">En su lugar, MED-V permite a los administradores administrar el área de trabajo de MED-V como lo haría en un equipo físico, lo que permite que las aplicaciones y las actualizaciones se distribuyan sin la complejidad de una infraestructura de MED-V dedicada.</span><span class="sxs-lookup"><span data-stu-id="626b6-129">Instead, MED-V enables administrators to manage the MED-V workspace as they would a physical computer, by letting applications and updates be distributed without the complexity of a dedicated MED-V infrastructure.</span></span>
