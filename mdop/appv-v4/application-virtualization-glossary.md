# Glosario de virtualización de aplicaciones


<table>
<colgroup>
<col width="50%" />
<col width="50%" />
</colgroup>
<thead>
<tr class="header">
<th align="left">Término</th>
<th align="left">Definición</th>
</tr>
</thead>
<tbody>
<tr class="odd">
<td align="left"><p>actualización activa</p></td>
<td align="left"><p>Una actualización que permite agregar una nueva versión de una aplicación a un servidor de administración de App-V o de streaming sin afectar a los usuarios que actualmente ejecutan la aplicación.</p></td>
</tr>
<tr class="even">
<td align="left"><p>Raíz de origen de la aplicación</p></td>
<td align="left"><p>Referencia a una clave del registro que permite reemplazar el código base del OSD para el elemento HREF (por ejemplo, la ubicación de origen). Este valor del registro permite que un administrador o sistema ESD garantice la carga de la aplicación según un esquema de administración de topología planificado.</p></td>
</tr>
<tr class="odd">
<td align="left"><p>Unidad de virtualización de aplicaciones</p></td>
<td align="left"><p>La unidad de cliente de aplicación virtual predeterminada (Q:) desde el que se ejecutan las aplicaciones con secuencia.</p></td>
</tr>
<tr class="even">
<td align="left"><p>Informes de virtualización de aplicaciones</p></td>
<td align="left"><p>Información de aplicaciones virtuales recopilada para el análisis de datos. Los datos se recopilan para el ensamblado de vistas personalizadas e interpretación.</p></td>
</tr>
<tr class="odd">
<td align="left"><p>Parámetro de configuración de carga automática</p></td>
<td align="left"><p>Parámetro de configuración de la Directiva de tiempo de ejecución del cliente que permite que el bloque de características secundario de una aplicación virtualizada se transmita automáticamente al cliente en segundo plano.</p></td>
</tr>
<tr class="even">
<td align="left"><p>bifurcar un paquete</p></td>
<td align="left"><p>Para actualizar un paquete de aplicación secuencial existente y ejecutarlo en paralelo con el paquete de aplicación secuencial original.</p></td>
</tr>
<tr class="odd">
<td align="left"><p>Composición de conjunto dinámico</p></td>
<td align="left"><p>Una característica de virtualización de aplicaciones que permite a un paquete de aplicaciones virtuales permitir que los complementos dependientes o los paquetes de middleware usen el entorno virtual. Esta característica permite que los complementos y los paquetes de middleware usen la configuración del registro del paquete principal, y los paquetes se comportan e interactúan entre sí, de la misma manera que si se instalaron de forma local en un equipo con micrófono.</p></td>
</tr>
<tr class="even">
<td align="left"><p>Archivo ICO</p></td>
<td align="left"><p>Archivo del icono en el escritorio del cliente que se usa para iniciar una aplicación de secuencia.</p></td>
</tr>
<tr class="odd">
<td align="left"><p>Raíz de origen del icono</p></td>
<td align="left"><p>Referencia a una clave del registro que permite la especificación de una ubicación de origen para la recuperación de iconos para un paquete de aplicación de secuencia durante la publicación. Las raíces de origen de icono solo admiten formatos UNC (no direcciones URL).</p></td>
</tr>
<tr class="even">
<td align="left"><p>Directorio de instalación</p></td>
<td align="left"><p>El directorio en el que el instalador del secuenciador de virtualización de la aplicación coloca los archivos.</p></td>
</tr>
<tr class="odd">
<td align="left"><p>Cliente de escritorio de Microsoft Application Virtualization</p></td>
<td align="left"><p>Una aplicación que reside en un escritorio de un equipo basado en Windows y que se comunica y se autentica con el servidor de aplicaciones virtual de Microsoft System Center para recibir el código de la aplicación y permitir que una aplicación secuenciada se ejecute de forma local.</p></td>
</tr>
<tr class="even">
<td align="left"><p>Microsoft Application Virtualization para servicios de Terminal Server</p></td>
<td align="left"><p>La característica de virtualización de aplicaciones, incluidos los componentes de cliente y servidor, que se ejecutan en un entorno de servicios de Terminal Server.</p></td>
</tr>
<tr class="odd">
<td align="left"><p>Consola de administración de Microsoft Application Virtualization</p></td>
<td align="left"><p>Complemento centralizado para la consola de administración de Microsoft, que se usa para administrar el sistema de administración de virtualización de aplicaciones.</p></td>
</tr>
<tr class="even">
<td align="left"><p>Sistema de administración de Microsoft Application Virtualization</p></td>
<td align="left"><p>Una implementación específica de la plataforma de virtualización de aplicaciones que incluye todos los componentes administrados por un único almacén de datos.</p></td>
</tr>
<tr class="odd">
<td align="left"><p>Servicio Web de administración de Microsoft Application Virtualization</p></td>
<td align="left"><p>Un servicio que actúa como intermediario entre la consola de administración de la virtualización de aplicaciones y el almacén de datos. En primer lugar, autentica a sus usuarios y, a continuación, permite a los administradores autorizados manipular datos en el almacén de datos.</p></td>
</tr>
<tr class="even">
<td align="left"><p>Plataforma de virtualización de aplicaciones de Microsoft</p></td>
<td align="left"><p>Nombre de los productos de Microsoft que se usan para crear, almacenar, distribuir y ejecutar aplicaciones virtualizadas.</p></td>
</tr>
<tr class="odd">
<td align="left"><p>Microsoft Application Virtualization Sequencer</p></td>
<td align="left"><p>Aplicación que supervisa y registra el proceso de instalación y configuración de las aplicaciones para que una aplicación se pueda secuenciar y ejecutar en el entorno virtual.</p></td>
</tr>
<tr class="even">
<td align="left"><p>Cliente de servicios de Terminal Server de Microsoft Application Virtualization</p></td>
<td align="left"><p>Una aplicación que reside en un terminal Server y que se comunica y se autentica con el servidor de aplicaciones virtual de Microsoft para recibir el código de la aplicación y permitir que una aplicación secuenciada se ejecute de forma local.</p></td>
</tr>
<tr class="odd">
<td align="left"><p>Microsoft System Center Application Virtualization Management Server</p></td>
<td align="left"><p>Uno de los dos tipos de servidor de Application Virtualization desde el que se puede transmitir un paquete de aplicaciones secuenciado. Además de transmitir paquetes de aplicaciones virtuales, Application Virtualization Management Server ofrece otros servicios, como la publicación, la administración, los informes, etc.</p></td>
</tr>
<tr class="even">
<td align="left"><p>Servidor de streaming de Microsoft System Center Application Virtualization</p></td>
<td align="left"><p>Uno de los dos tipos de servidor de Application Virtualization desde el que se puede transmitir un paquete de aplicaciones secuenciado. El servidor de transmisión transmite las aplicaciones solo a los equipos cliente y no ofrece otros servicios, como la publicación, la administración, los informes, etc.</p></td>
</tr>
<tr class="odd">
<td align="left"><p>Archivo descriptor de software (OSD) abierto</p></td>
<td align="left"><p>Un archivo basado en XML que indica al cliente cómo recuperar la aplicación secuenciada de la aplicación de virtualización/servidor de streaming, y cómo ejecutar la aplicación secuenciada en su entorno virtual.</p></td>
</tr>
<tr class="even">
<td align="left"><p>Raíz de origen OSD</p></td>
<td align="left"><p>Referencia a una clave del registro que permite la especificación de una ubicación de origen para la recuperación de archivos OSD para un paquete de aplicación durante la publicación. Las raíces de origen OSD solo admiten formatos UNC (no direcciones URL).</p></td>
</tr>
<tr class="odd">
<td align="left"><p>directorio raíz del paquete</p></td>
<td align="left"><p>El directorio en el equipo de secuencia en el que se instalan los archivos para el paquete de aplicación de secuencia. Este directorio también existe prácticamente en el equipo al que se transmitirá una aplicación de secuencia.</p></td>
</tr>
<tr class="even">
<td align="left"><p>bloque de características principal</p></td>
<td align="left"><p>El contenido mínimo de un paquete de la aplicación que es necesario para que se ejecute una aplicación. El contenido del bloque de características principal se identifica durante la fase de aplicación de las secuencias y generalmente consta del contenido de las características de aplicación más usadas.</p></td>
</tr>
<tr class="odd">
<td align="left"><p>bloque de características secundario</p></td>
<td align="left"><p>El resto del paquete de la aplicación que no está contenido en el bloque de características principal. Este contenido se transmite al cliente a petición a medida que se usan las características de la aplicación.</p></td>
</tr>
<tr class="even">
<td align="left"><p>aplicación de secuencia</p></td>
<td align="left"><p>Una aplicación que ha sido supervisada por el secuenciador, dividida en bloques de características principales y secundarios, se transmite a un equipo que ejecuta el cliente de servicios de Terminal Server de Microsoft Application Virtualization o el cliente de escritorio de Microsoft Application Virtualization, y se puede ejecutar dentro de su propio entorno virtual.</p></td>
</tr>
<tr class="odd">
<td align="left"><p>paquete de aplicación de secuencia</p></td>
<td align="left"><p>Los archivos que componen una aplicación virtual y permiten que se ejecute una aplicación virtual. Estos archivos se crean después de la secuenciación e incluyen específicamente los archivos. OSD,. SFT,. SPRJ y. ico.</p></td>
</tr>
<tr class="even">
<td align="left"><p>128</p></td>
<td align="left"><p>El proceso de creación de un paquete de aplicación mediante Application Virtualization Sequencer. En este proceso, se supervisa una aplicación, se configuran los métodos abreviados y se crea un paquete de aplicación de secuencia que contiene los archivos. OSD,. SFT,. SPRJ e. ico.</p></td>
</tr>
<tr class="odd">
<td align="left"><p>equipo de secuencias</p></td>
<td align="left"><p>El equipo usado para realizar la secuenciación y crear un paquete de aplicación de secuencia.</p></td>
</tr>
<tr class="even">
<td align="left"><p>Asistente de secuenciamiento</p></td>
<td align="left"><p>Asistente paso a paso que guía al usuario a través de la secuenciación de una aplicación, incluida la configuración del paquete, la instalación de la aplicación o las aplicaciones que se van a secuenciar y la secuenciación del paquete de aplicación para la transmisión.</p></td>
</tr>
<tr class="odd">
<td align="left"><p>SFT</p></td>
<td align="left"><p>Un archivo que contiene una o más aplicaciones secuenciadas que el secuenciador ha empaquetado en bloques de transmisión por secuencias, así como la información de entrega asociada. El archivo SFT se almacena en cada servidor que debe transmitir las aplicaciones empaquetadas a un cliente.</p></td>
</tr>
<tr class="even">
<td align="left"><p>SFTMIME</p></td>
<td align="left"><p>Una interfaz de línea de comandos para configurar y administrar aplicaciones, asociaciones de tipos de archivo y servidores de configuración de escritorio administrados por los clientes de escritorio y servicios de Terminal Server de Application Virtualization.</p></td>
</tr>
<tr class="odd">
<td align="left"><p>Archivo SPRJ</p></td>
<td align="left"><p>Archivo de proyecto SPRJ basado en XML en el que el secuenciador almacena los elementos de exclusión y la información de los elementos de análisis. El archivo SPRJ se usa intensamente en la creación de registros de aplicaciones y también durante una actualización a un paquete.</p></td>
</tr>
<tr class="even">
<td align="left"><p>aplicación virtual</p></td>
<td align="left"><p>Una aplicación empaquetada por el secuenciador para que se ejecute en un entorno virtual independiente. El entorno virtual contiene la información necesaria para ejecutar la aplicación en el cliente sin instalar la aplicación de forma local.</p></td>
</tr>
<tr class="odd">
<td align="left"><p>COM virtual</p></td>
<td align="left"><p>El subsistema que administra los objetos COM creados por procesos de aplicaciones que se ejecutan en un entorno virtual y evita conflictos con los mismos objetos creados fuera del entorno virtual.</p></td>
</tr>
<tr class="even">
<td align="left"><p>directorio virtual</p></td>
<td align="left"><p>Un directorio opaco en el que solo se pueden ver los archivos y subdirectorios definidos en el paquete de aplicación o creados mediante la interacción con una aplicación en un entorno virtual. Los archivos de un directorio local con nombres idénticos no son visibles para la aplicación.</p></td>
</tr>
<tr class="odd">
<td align="left"><p>entorno virtual</p></td>
<td align="left"><p>Un contenedor en tiempo de ejecución que define los recursos disponibles para los procesos de la aplicación iniciados desde un paquete de aplicación de secuencia.</p></td>
</tr>
<tr class="even">
<td align="left"><p>archivo virtual</p></td>
<td align="left"><p>Un nombre de archivo dentro del entorno virtual que está asignado a una ubicación de destino alternativa. Aparece un archivo virtual junto a otros archivos en el directorio contenedor, independientemente de si el directorio es virtual o local.</p></td>
</tr>
<tr class="odd">
<td align="left"><p>sistema de archivos virtual</p></td>
<td align="left"><p>Subsistema que intercepta y redirige las solicitudes del sistema de archivos de los procesos de la aplicación que se ejecutan en un entorno virtual. Estas solicitudes se procesan en función de los archivos virtuales y directorios definidos en el paquete de la aplicación y se crean o modifican mediante la interacción con una aplicación virtual.</p></td>
</tr>
<tr class="even">
<td align="left"><p>registro virtual</p></td>
<td align="left"><p>Subsistema que intercepta y redirige las solicitudes de registro de claves y valores de los procesos de aplicaciones que se ejecutan en un entorno virtual. El redireccionamiento se basa en la información del registro definida en el paquete de la aplicación y creada o modificada mediante la interacción con una aplicación virtual.</p></td>
</tr>
<tr class="odd">
<td align="left"><p>servicios virtuales</p></td>
<td align="left"><p>Subsistema que actúa como administrador de control de servicios (SCM) para los servicios que se ejecutan en un entorno virtual.</p></td>
</tr>
</tbody>
</table>
